Memact

Apps can remember only what the user approves.

Memact gives developers a consent-first way to use helpful memory. Users see the app, scopes, activity categories, and data disclosure before access is allowed.

For users

You approve or cancel each app. Data Transparency explains what the app plans to use, including captured fields, evidence, summaries, graph packets, retention, and revocation.

For developers

Register an app, choose narrow scopes and categories, send users through consent, keep the raw API key on your server, and verify access before using Memact output.

No blanket graph dump

Apps receive only approved output. Raw graph access is separate, sensitive, and must be explicitly allowed.

Categories are boundaries

Activity categories limit where an app can work, such as research, news, AI conversations, developer work, media, or documents.

Basics

What is Memact?

Memact lets apps remember useful context from user-approved activity without giving them blanket access.

What is the consent flow?

The consent flow shows the app, requested scopes, activity categories, safety boundaries, and a Data Transparency link before the user approves.

What is Data Transparency?

Data Transparency is the page beside consent that explains actual captured fields, evidence cards, summaries, graph packets, retention, and how to revoke future access.

What does an API key do?

An API key identifies the app. The app still needs a user connection id and approved scopes before Memact allows access.

Developer flow

How do I use Memact in an app?
  1. Register the app in the Memact portal.
  2. Choose the smallest scopes and activity categories needed.
  3. Add a Connect Memact button that opens `/connect` with app context.
  4. Link `/DataTransparency` with the same app context beside consent.
  5. Store the returned `connection_id` for the signed-in user.
  6. Keep the raw API key on your server, not in client code.
  7. Verify the API key, connection id, and required scopes before using Memact output.
What is a schema packet?

A schema packet is a memory object that can include evidence, content units, nodes, edges, and a summary.

What should apps avoid?

Apps should not request more access than they need, hide collection, sell raw memory, use Memact for manipulation, or use it for sensitive eligibility decisions.
