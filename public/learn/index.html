Learn more about Memact

A simple guide to how Memact works, what apps can see, and how you stay in control.

Basics

What is Memact?v

Memact helps apps understand digital activity only after the user approves the right activity and memory permissions.

Where does Memact run?v

Apps use a small Memact client. Evidence can come from the browser extension and, if enabled, a local helper. Memact turns approved evidence into context, schemas, and memory; apps do not receive a blanket activity feed.

Does an app get my whole memory graph?v

No. Apps only get the context, summaries, evidence, or graph objects allowed by the permissions and categories you approve. Raw graph access is separate and sensitive.

How does consent work?v

Consent shows what an app wants Memact to understand before anything is connected. You can approve or cancel, and Data Transparency lets you review the evidence, schemas, memory objects, and context first.

What are activity categories?v

They limit which part of your activity Memact can understand for that app, like research pages, news, AI conversations, developer activity, or media.

Activity and memory

What activity can be used?v

Memact can use approved evidence like page titles, URLs, domains, timestamps, selected text, page text, PDF text, captions, transcripts, image context, and optional app or window context from the local helper. That evidence exists to produce understanding, not to sell a raw capture feed.

What does Memact turn activity into?v

Memact turns approved activity into context objects: events, sessions, evidence cards, summaries, schema packets, graph links, and patterns that can become memory. An app only sees what its saved permissions allow.

What does an app receive?v

Only scoped understanding allowed by the app key, your consent, selected scopes, activity categories, and authorized origin. That can include compact summaries, evidence cards, schema packets, memory objects, graph objects when approved, and status signals.

What is blocked?v

Memact skips sensitive pages like banking, payments, checkout, billing, passwords, logins, OTPs, private inboxes, direct messages, medical portals, and private account or admin pages before they become memory evidence.

Advanced

How do I use the Memact API in an app?v

Use Memact as a permissioned understanding and memory layer, not as a raw capture feed.

  1. Register your app in Memact and choose the smallest scopes and activity categories your feature needs.
  2. Add a Connect Memact button in your app.
  3. Put a Data Transparency link beside that consent flow.
  4. After approval, store the returned connection_id for the signed-in user in your app.
  5. Keep the raw Memact API key in server environment config.
  6. Your backend verifies the API key, connection id, required scopes, and activity categories before requesting context.
  7. Use only the approved understanding Memact returns.
Where should the Memact API key live in code?v

Treat the raw Memact API key like a server-side secret. This is the private app key that starts with mka_ and is created in the Memact portal. Do not hard-code it into browser bundles, mobile apps, public repos, logs, shared prompts, or user-facing settings panels.

Do I need a Supabase public key?v

No. App developers configure only the private Memact app key on their own backend. Supabase keys belong to Memact's access infrastructure, not customer apps.

Is a Data Transparency page required?v

Yes. Any app asking users to consent through Memact should expose Data Transparency next to the consent flow. It must explain the evidence fields, context objects, summaries, graph packets, retention, and revocation path. Categories alone are not enough.

What is a schema packet?v

A schema packet is Memact's understanding envelope: evidence, content units, nodes, edges, inferred structure, and a short summary that later layers can decide to keep as memory.

What is not allowed?v

Apps should only request what they need. Hidden monitoring, selling raw activity or memory, manipulation, and sensitive decision-making are not allowed.
