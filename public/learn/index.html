Memact

Permissioned memory infrastructure for apps.

Apps ask for memory access. You choose what they get.

What is Memact?

Memact is infrastructure that lets apps create permissioned memory from your digital activity.

How does app access work?

Developers register apps, choose activity categories, request scoped permissions, and create API keys. Users approve the access before an app connects.

Does an app get my whole memory graph?

No. Apps only get the permissions and activity categories you approve. Raw graph access is separate and sensitive.

What are activity categories?

Activity categories limit where an app can work, such as research pages, news, AI conversations, or developer activity.

Basics

What is permissioned memory?

Permissioned memory means an app can only use memory capabilities that were explicitly approved for that app.

What does Connect App do?

Connect App shows what an app wants before anything is connected. You approve or cancel.

What does an API key do?

An API key identifies an app. Memact checks the key against the user's approved scopes and categories before access is allowed.

Advanced

What is a schema packet?

A schema packet is Memact's internal memory unit: evidence, content, nodes, edges, and a short summary.

What should apps avoid?

Apps should request only what they need, explain their access clearly, and avoid using Memact outside the approved scopes and categories.

Memact is source-available infrastructure for permissioned memory access. This page explains the public concept; the access portal handles app registration, permissions, and API keys.
